Introduction

Everyone saw different interesting robots which can perform really interesting things. However does anyone really asked himself a question: "How?". Well, it is all because of two things: hardware and programm inside it. So, here we will see a programm aspect of robots. We are going to get to know to ROS.

ROS

What is ROS?

ROS stands for Robotic Operating System. It is designed to completely fullfil one's desire to control robots. It provides a set of package to control robot, so, we can just install it and run our robots as we want.

Why ROS?

ROS is highly developed robotic System, also because of its open-source philosophy it is easy to get into and a lot of projects are done not only by professional but by enthusiast too.

installation

Prepare remote PC first!

Well, to use ROS we need both: robot with ROS and remote PC (Host) with ROS. So, let us see how to prepare remote PC for ROS first.

It's recommended to use ROS in Ubuntu or Ubuntu based disributives.

  1. Download Ubuntu and install it.
  2. Install ROS on remote PC (further you will see a set of commands to instal ROS)
    1. $ sudo apt update
    2. $ sudo apt upgrade
    3. $ wget https://raw.githubusercontent.com/ROBOTIS-GIT/robotis_tools/master/install_ros_noetic.sh
    4. $ chmod 755 ./install_ros_noetic.sh
    5. $ bash ./install_ros_noetic.sh
  3. Install all dependencies.
    4. $ sudo apt-get install ros-noetic-joy ros-noetic-teleop-twist-joy \ ros-noetic-teleop-twist-keyboard ros-noetic-laser-proc \ ros-noetic-rgbd-launch ros-noetic-rosserial-arduino \ ros-noetic-rosserial-python ros-noetic-rosserial-client \ ros-noetic-rosserial-msgs ros-noetic-amcl ros-noetic-map-server \ ros-noetic-move-base ros-noetic-urdf ros-noetic-xacro \ ros-noetic-compressed-image-transport ros-noetic-rqt* ros-noetic-rviz \ ros-noetic-gmapping ros-noetic-navigation ros-noetic-interactive-markers
